打包,通过winscp上传文件,不要删除上一次发布的文件,等到下一次在发布的时候删除

,打开xshell,查看72村运行的进程,进程是第一个
ps -ef | grep 72cun

杀死进程
kill -9 8080

运行,并在运行时临时指定动态端口,方便nginx进行重新转发到改端口
nohup java -jar /72cun/72cun-1.3.3-SNAPSHOT.jar --spring.profiles.active=prod --server.port=8888 >>  /72cun/temp_8888.log 2>&1

查看输出日志
tail -f /72cun/temp_8888.log
